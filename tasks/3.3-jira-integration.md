# 3.3 Jira Integration

**Priorität:** MITTEL
**Aufwand:** 3 Tage
**Phase:** 3 — Enterprise-Integration

## Ziel

Bidirektionale Synchronisation zwischen Triologue Tasks und Jira Issues. Agent-Ergebnisse koennen als Jira-Tickets angelegt werden. Jira-Tickets koennen als Triologue-Tasks importiert werden.

## Hintergrund

Christians Anforderung: "gerne Jira". Viele Teams nutzen Jira fuer Projektmanagement. Wenn Triologue-Tasks mit Jira synchronisiert werden, muessen Teams nicht zwischen zwei Systemen wechseln.

## Anforderungen

### Jira Cloud API

- OAuth 2.0 (3LO) fuer Jira Cloud
- Scopes: `read:jira-work`, `write:jira-work`
- Base URL: `https://{instance}.atlassian.net/rest/api/3/`

### Sync-Richtungen

**Triologue → Jira (Push):**
- Agent schliesst Task ab → Jira Issue wird erstellt/aktualisiert
- Task-Status Mapping: todo→To Do, in_progress→In Progress, in_review→In Review, done→Done
- Attachments werden mit synchronisiert

**Jira → Triologue (Pull):**
- Neues Jira Issue → Triologue Task wird erstellt
- Status-Aenderung in Jira → Triologue Task wird aktualisiert
- Webhook-basiert (Jira Webhooks)

### Mapping-Konfiguration

Pro Projekt:
- Jira Project Key (z.B. "PP")
- Issue Type Mapping (Task → Story, Bug, etc.)
- Status Mapping (konfigurierbar)
- Sync-Richtung: Push only, Pull only, Bidirektional

### Conflict Resolution

- Last-Write-Wins mit Timestamp-Vergleich
- Bei Konflikten: Audit Log Eintrag + Benachrichtigung

## Abhaengigkeiten

- Atlassian Developer Account
- Jira Cloud Instance
- Task 3.4 (OAuth Token Management)

5. **`server/prisma/schema.prisma`** — Sync-Mapping Tabelle (Triologue Task ID ↔ Jira Issue ID)
6. **Admin UI** — Jira-Verbindung konfigurieren, Mapping definieren
