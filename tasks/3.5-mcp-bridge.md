# 3.5 MCP Bridge (Custom Connectors)

**Priorität:** MITTEL
**Aufwand:** 3 Tage
**Phase:** 3 — Enterprise-Integration

## Ziel

Admins koennen Custom Connectors ueber das Model Context Protocol (MCP) installieren. Triologue entdeckt die bereitgestellten Tools automatisch und registriert sie als Actions in der Action Registry.

## Hintergrund

YAML-Connectors decken 90% der Faelle ab (SaaS-Dienste mit REST API + OAuth). Fuer die restlichen 10% — interne Systeme, Legacy-Software, proprietaere Datenbanken — brauchen Kunden die Moeglichkeit, eigene Connectors zu bauen. MCP ist der sich etablierende Standard dafuer.

## Was ist MCP?

Model Context Protocol (modelcontextprotocol.io) ist ein offener Standard:
- **MCP Server:** Exponiert Tools (Funktionen) ueber ein standardisiertes Protokoll
- **MCP Client:** Entdeckt und ruft diese Tools auf
- **Transport:** stdio oder SSE (Server-Sent Events)

Beispiel: Ein Kunde hat ein internes ERP. Er baut einen MCP Server davor der `erp.getInvoice` und `erp.createOrder` als Tools exponiert. Triologue entdeckt diese Tools und macht sie als Actions verfuegbar.

## Anforderungen

### MCP Server Registration

```
POST /api/admin/connectors/mcp
{
  "name": "Internes ERP",
  "transport": "sse",           // "sse" oder "stdio"
  "url": "https://erp-mcp.internal.company.com/mcp",
  "apiKey": "optional-auth-key"
}
```

### Tool Discovery

Nach Registration:
1. Triologue verbindet sich zum MCP Server
2. Ruft `tools/list` auf (MCP Standard)
3. Bekommt Liste der verfuegbaren Tools mit Schemas
4. Registriert jedes Tool als Action in der Action Registry

```typescript
// MCP Server antwortet auf tools/list:
{
  "tools": [
    {
      "name": "erp.getInvoice",
      "description": "Rechnung aus ERP laden",
      "inputSchema": {
        "type": "object",
        "properties": {
          "invoiceId": { "type": "string" }
        }
      }
    }
  ]
}

// → Wird registriert als Action:
{
  "id": "mcp:erp.getInvoice",
  "name": "Rechnung aus ERP laden",
  "type": "connector",
  "connectorId": "mcp:internes-erp",
  "url": "/api/connectors/mcp/internes-erp/actions/erp.getInvoice"
}
```

### Proxy fuer MCP

Agent ruft Triologue Proxy auf → Proxy leitet an MCP Server weiter:

```
Agent: POST /api/connectors/mcp/internes-erp/actions/erp.getInvoice
       { "invoiceId": "INV-2026-001" }
       
Triologue: tools/call → MCP Server
           { "name": "erp.getInvoice", "arguments": { "invoiceId": "INV-2026-001" } }
           
MCP Server: → ERP Datenbank → Response

Triologue: Response normalisieren → Agent
```

### Health Check

- Periodisch pruefen ob MCP Server erreichbar ist
- Bei Ausfall: Actions aus Registry entfernen, Admin benachrichtigen
- Bei Reconnect: Tools neu discovern (koennen sich geaendert haben)
