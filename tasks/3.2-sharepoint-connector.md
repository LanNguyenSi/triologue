# 3.2 SharePoint Connector

**Priorität:** HOCH
**Aufwand:** 2 Tage (reduziert dank Connector Framework)
**Phase:** 3 — Enterprise-Integration

**HINWEIS:** Baut auf dem Connector Framework (Task 3.1) auf.
Hauptarbeit ist die YAML-Definition + ggf. Custom Plugin fuer Binary-Handling.

## Ziel

Agents koennen Dateien aus SharePoint lesen und Ergebnisse in SharePoint ablegen. Konfigurierbar pro Projekt: welcher SharePoint-Ordner ist verknuepft.

## Hintergrund

Christians Anforderung: "Ablage via SharePoint". Viele publicplan-Kunden nutzen SharePoint als zentrale Dokumentenablage. Wenn Agent-Ergebnisse automatisch dort landen, ist die Integration in bestehende Workflows nahtlos.

## Anforderungen

### OAuth Flow

- Azure AD App Registration mit Graph API Permissions
- Scopes: `Files.ReadWrite.All`, `Sites.Read.All`
- OAuth 2.0 Authorization Code Flow (Admin Consent)
- Token-Management: Access Token + Refresh Token persistent speichern

### Actions im Task Runtime Context

```json
{
  "actions": {
    "sharepoint.read": {
      "method": "GET",
      "url": "https://graph.microsoft.com/v1.0/drives/{driveId}/items/{itemId}/content",
      "auth": "managed",
      "note": "Platform setzt OAuth Token automatisch"
    },
    "sharepoint.upload": {
      "method": "PUT",
      "url": "https://graph.microsoft.com/v1.0/drives/{driveId}/root:/{path}:/content",
      "auth": "managed"
    },
    "sharepoint.list": {
      "method": "GET",
      "url": "https://graph.microsoft.com/v1.0/drives/{driveId}/root/children"
    }
  }
}
```

### Projekt-Konfiguration

Pro Projekt konfigurierbar:
- SharePoint Site URL
- Drive / Ordner-Pfad
- Lese-Rechte (Input-Ordner fuer Dokumente)
- Schreib-Rechte (Output-Ordner fuer Ergebnisse)

### Proxy-Ansatz (empfohlen)

Agent ruft nicht direkt Graph API auf. Stattdessen:
1. Agent nutzt Triologue-Actions (`sharepoint.read`, `sharepoint.upload`)
2. Triologue-Backend proxied die Anfrage an Graph API mit dem gespeicherten OAuth Token
3. Agent sieht nie einen OAuth Token

Vorteil: Token-Management zentral, Agent braucht keine Microsoft-Kenntnisse.
