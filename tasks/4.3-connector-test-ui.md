# 4.3 Connector Test & Debug UI

**Priorität:** MITTEL
**Aufwand:** 1 Tag
**Phase:** 4 — Connector UI
**Abhängigkeit:** 4.0, 4.1

## Ziel

Admin kann Connector-Actions direkt in der UI testen, ohne Agent oder API-Client. Debug-Tool für Setup und Troubleshooting.

## Hintergrund

Wenn ein Admin SharePoint verbindet, will er sofort testen ob es funktioniert. Aktuell müsste er curl benutzen. Eine Test-UI senkt die Einstiegshürde massiv.

## Anforderungen

### Test-Console auf Connector Detail Page

Seite: `/admin/connectors/:connectorId/test`

1. **Action-Auswahl:** Dropdown mit allen Actions des Connectors
2. **Input-Formular:** Dynamisch generiert aus der Action-Definition (input schema)
   - z.B. SharePoint List: driveId (text), path (text, default "/")
   - z.B. Jira Create: cloudId, projectKey, summary, description
3. **"Ausführen" Button:** Sendet Request über Proxy (als Admin, nicht als Agent)
4. **Response-Anzeige:**
   - Status Code (grün/rot)
   - Response Body (JSON formatiert, collapsible)
   - Dauer (ms)
   - Fehler-Details wenn fehlgeschlagen

### Admin-Proxy Endpoint

```
POST /api/admin/connectors/:connectorId/test/:actionId
```

- Gleiche Logik wie Agent-Proxy, aber Admin-Auth statt Agent-Auth
- Kein Permission-Check (Admin darf alles testen)
- Audit-Log mit action `connector.test`

## Akzeptanzkriterien

- [ ] Admin kann jede Action eines Connectors testen
- [ ] Input-Formular wird dynamisch aus YAML-Schema generiert
- [ ] Response wird formatiert angezeigt (JSON pretty-print)
- [ ] Fehler sind verständlich ("Token abgelaufen", "SharePoint Site nicht gefunden")
- [ ] Test-Aufrufe werden im Audit Log geloggt
