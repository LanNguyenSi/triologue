# 3.0 OAuth Token Management

**Priorität:** KRITISCH (Fundament fuer alle Connectors)
**Aufwand:** 2 Tage
**Phase:** 3 — Enterprise-Integration (MUSS als erstes in Phase 3 gebaut werden)

## Ziel

Zentrale, sichere Verwaltung von OAuth Tokens fuer externe Integrationen. Tokens werden verschluesselt gespeichert, automatisch refreshed und nie an Agents weitergegeben.

## Hintergrund

Jede Enterprise-Integration (SharePoint, Jira, Teams, etc.) braucht OAuth Tokens. Ohne zentrale Token-Verwaltung muesste jeder Connector seine eigene Auth-Logik implementieren. Token Management ist das Fundament auf dem ALLE Connectors aufbauen.

## Warum zuerst?

Alle anderen Phase-3 Tasks haengen davon ab:
- 3.1 Connector Framework → braucht Token Storage
- 3.2 SharePoint → braucht Microsoft OAuth Token
- 3.3 Jira → braucht Atlassian OAuth Token
- 3.4 MS Teams Channel → braucht Microsoft Bot Token

## Anforderungen

### Prisma Schema

```prisma
model IntegrationToken {
  id            String   @id @default(cuid())
  provider      String                        // "microsoft", "atlassian", "google"
  scope         String                        // "graph", "jira", "gmail"
  accessToken   String                        // Verschluesselt (AES-256-GCM)
  refreshToken  String?                       // Verschluesselt
  expiresAt     DateTime
  tenantId      String?                       // Azure Tenant, Jira Cloud ID, etc.
  metadata      Json     @default("{}")       // Provider-spezifische Infos
  createdBy     String                        // Admin der die Integration eingerichtet hat
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  
  creator       User     @relation(fields: [createdBy], references: [id])
  
  @@unique([provider, scope, tenantId])
  @@map("integration_tokens")
}
```

### Token Encryption

- AES-256-GCM Verschluesselung vor dem Speichern
- Encryption Key aus Environment Variable (`INTEGRATION_ENCRYPTION_KEY`)
- Decryption nur im Backend, nie in API-Responses
- Separate IV pro Token (nicht global)

### Auto-Refresh Service

```typescript
// server/src/services/tokenManager.ts

class TokenManager {
  // Token holen (decrypted, refreshed wenn noetig)
  async getToken(provider: string, scope: string): Promise<string>;
  
  // Token speichern (encrypted)
  async storeToken(provider: string, scope: string, tokens: OAuthTokens): Promise<void>;
  
  // Token loeschen
  async revokeToken(provider: string, scope: string): Promise<void>;
  
  // Alle Tokens pruefen, abgelaufene refreshen
  async refreshExpiring(): Promise<void>;  // Laeuft als Background Job
}
```

- Background Job alle 5 Minuten: Tokens pruefen die in <10 Min ablaufen
- Bei Refresh-Fehler: Status auf "error" setzen, Admin benachrichtigen via Inbox
- Bei permanentem Fehler (Refresh Token invalid): Token deaktivieren, Integration als "disconnected" markieren

### OAuth Flow Endpoints

```
GET  /api/admin/integrations/{provider}/connect    → Redirect zu OAuth Provider
GET  /api/admin/integrations/{provider}/callback   → OAuth Callback, Token speichern
GET  /api/admin/integrations                       → Liste aller Integrationen + Status
DELETE /api/admin/integrations/{provider}           → Integration trennen, Token loeschen
```

### Berechtigung

- Nur Admins koennen Integrationen verbinden/trennen
- Token-Inhalte sind nie in API-Responses sichtbar
- Agents nutzen Tokens nur indirekt ueber den Proxy Layer (siehe Task 3.1)
