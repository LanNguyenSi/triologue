# 2.5 Agent-Konfiguration UI

**Priorität:** MITTEL
**Aufwand:** 2 Tage
**Phase:** 2 — Workflow-Reife

## Ziel

Admin-UI um Agent-Verhalten pro Agent zu konfigurieren: erlaubte Aktionen, Nachrichtenfrequenz, Proaktivitaet.

## Hintergrund

Koala-Paper (IBM, IUI 2025) zeigt: Nutzer wollen Kontrolle ueber Agent-Verhalten. Aktuell ist Agent-Verhalten nur ueber System-Prompts und Memories steuerbar — das reicht nicht fuer eine Enterprise-Demo.

## Anforderungen

### Agent Settings Seite

Erreichbar ueber Admin → Agents → [Agent] → Settings

### Konfigurierbare Felder

```typescript
interface AgentConfig {
  // Kommunikation
  messageFrequency: 'low' | 'medium' | 'high';    // Wie oft darf der Agent posten
  proactivity: 'reactive' | 'balanced' | 'proactive';  // Nur bei @mention vs. eigeninitiativ
  maxMessagesPerMinute: number;                     // Rate Limit (default: 5)
  
  // Aktionen
  canUploadAttachments: boolean;                    // default: true
  canCreateTasks: boolean;                          // default: false
  canUpdateTaskStatus: boolean;                     // default: true
  canDeleteMessages: boolean;                       // default: false
  
  // Inhalt
  suppressMetaReflections: boolean;                 // default: true (keine "Ich habe gelernt dass...")
  maxResponseLength: number;                        // default: 4000
  language: string;                                 // default: 'de'
}
```

### Persistierung

- In `agentToken` Tabelle: neues `config` JSONB Feld
- Oder: separate `agent_config` Tabelle

### Enforcement

Config wird im Webhook-Payload an den Agent mitgeliefert. Der Agent SOLLTE sich daran halten.
Zusaetzlich: Server-seitige Enforcement wo moeglich:
- `maxMessagesPerMinute` → Rate Limiting im Message-Endpoint
- `canUploadAttachments` → 403 wenn false
- `canCreateTasks` → 403 wenn false

### Frontend

- Settings-Formular mit Toggles und Dropdowns
- Live-Preview: "Mit diesen Einstellungen wird der Agent..."
- Reset-Button fuer Defaults

## Dateien die geaendert/erstellt werden muessen

1. **`server/prisma/schema.prisma`** — config Feld auf AgentToken oder neue Tabelle
2. **`server/src/routes/agents.ts`** — Config CRUD Endpoints
3. **`server/src/routes/rooms.ts`** — Config im Message-POST enforcing (Rate Limit)
4. **`client/src/pages/AdminPage.tsx`** oder neue `AgentConfigPage.tsx` — Settings UI
5. **Prisma Migration**
