# 3.1 MS Teams Bot Connector

**Priorität:** HOCH
**Aufwand:** 5 Tage
**Phase:** 3 — Enterprise-Integration

## Ziel

Triologue-Agents sind via @mention in Microsoft Teams ansprechbar. Nachrichten werden bidirektional synchronisiert: Teams ↔ Triologue Room.

## Hintergrund

Christian H. (GL Projekte & Loesungen) hat als Zielbild "konfigurierbare Agenten in MS Teams via @mention" beschrieben. Teams ist das primaere Kommunikationstool bei publicplan und vielen GovTech-Kunden.

## Anforderungen

### Architektur

```
MS Teams Channel
    ↕ (Bot Framework / Graph API)
Teams Bot Service (Node.js)
    ↕ (REST API)
Triologue API
    ↕ (SSE/WebSocket)
Triologue Agents
```

### Bot Registration

- Azure Bot Service Registration
- App ID + Secret
- Permissions: Team.ReadBasic.All, ChannelMessage.Read.All, ChannelMessage.Send

### Nachrichtenfluss

**Teams → Triologue:**
1. User schreibt @AgentName in Teams Channel
2. Bot Framework Webhook empfaengt Nachricht
3. Bot Service mappt Teams User → Triologue User (oder erstellt Guest)
4. Bot Service postet Nachricht in zugeordneten Triologue Room
5. Agent verarbeitet und antwortet im Triologue Room

**Triologue → Teams:**
1. Agent postet Antwort im Triologue Room
2. SSE Event an Bot Service
3. Bot Service postet Antwort in Teams Channel

### Mapping

- Teams Channel ↔ Triologue Room (1:1 oder konfigurierbar)
- Teams User ↔ Triologue User (auto-create als Guest oder manuelles Mapping)
- @mention in Teams → @mention in Triologue

### Einschraenkungen

- Attachments: Teams → Triologue (Download via Graph API, Upload via Triologue API)
- Attachments: Triologue → Teams (Upload via Graph API)
- Formatierung: Markdown ↔ Adaptive Cards Konvertierung
