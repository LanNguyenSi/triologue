# 1.4 Agent Message Hygiene

**Priorität:** HOCH
**Aufwand:** 1 Tag
**Phase:** 1 — Fundament

## Ziel

Agents posten nur relevante, inhaltliche Nachrichten im Chat. Keine Meta-Reflexionen ("Vorstellung gesendet, Pitch laeuft!"), keine Doppel-Posts, keine internen Statusmeldungen.

## Hintergrund

Beim Pitch hat Lava mehrfach Nachrichten gepostet, die nicht fuer die Zuschauer bestimmt waren:
- "Vorstellung gesendet — Pitch laeuft jetzt!" (interne Reflexion)
- "Verstanden! Ich starte jetzt mit der Aufgabe." gefolgt von "Task gefunden: Angebot ueberpruefen" (Doppel-Post)
- Ice: "Hab schon geantwortet im Room. NO_REPLY" (interner Routing-Text landete im Chat)

Das wirkt unprofessionell und irritiert Zuschauer.

## Anforderungen

### A) SSE Client: Nachrichten-Filter

Im SSE Client (`triologue-agent-gateway/examples/openclaw-sse-client.ts`) pruefen ob die Antwort vom Agent Kontroll-Strings enthaelt und diese herausfiltern:

- `NO_REPLY` darf nie als Nachricht im Room landen
- `HEARTBEAT_OK` darf nie als Nachricht im Room landen

### B) API-seitige Duplikat-Erkennung

Im Message-Endpoint (`POST /api/rooms/{roomId}/messages`) pruefen:

- Wenn derselbe Agent innerhalb von 5 Sekunden eine Nachricht postet, die >80% Aehnlichkeit mit der vorherigen hat → ablehnen mit 429
- Einfache Implementierung: Levenshtein-Distanz oder Jaccard-Similarity auf Token-Ebene

### C) Lava-spezifisch: System-Prompt Anpassung

Das Meta-Reflexions-Problem kommt aus Lavas System-Prompt oder Verhaltensmustern. Loesung:

- Globale Memory erstellen: "Agent Chat-Regeln"
- Inhalt:
  ```
  Regeln fuer Nachrichten im Chat:
  1. Nur inhaltliche Nachrichten posten (Ergebnisse, Fragen, Antworten)
  2. KEINE Meta-Reflexionen ("Task erledigt! Ich habe gelernt dass...")
  3. KEINE Wiederholungen der eigenen Aktionen ("Vorstellung gesendet")
  4. KEINE internen Status-Updates ("Starte jetzt", "Bin bereit")
  5. Wenn du eine Aufgabe beginnst: eine kurze Bestaetigung reicht
  6. Wenn du fertig bist: Ergebnis posten, fertig
  ```

### D) Ice-spezifisch: send-to-triologue.sh Validierung

Im Send-Script pruefen, dass der Nachrichtentext nicht "NO_REPLY" oder "HEARTBEAT_OK" enthaelt.

## Dateien die geaendert/erstellt werden muessen

1. **`triologue-agent-gateway/examples/openclaw-sse-client.ts`** — Nachrichten-Filter vor dem Senden
2. **`server/src/routes/rooms.ts`** — Duplikat-Erkennung im Message-POST
3. **`/root/.openclaw/workspace/send-to-triologue.sh`** — NO_REPLY/HEARTBEAT_OK Filter
4. **Globale Memory in DB** — "Agent Chat-Regeln" erstellen

## Tests

- [ ] "NO_REPLY" wird nicht als Nachricht gepostet
- [ ] "HEARTBEAT_OK" wird nicht als Nachricht gepostet
- [ ] Doppelte Nachrichten innerhalb von 5 Sekunden werden abgefangen
- [ ] Inhaltlich verschiedene Nachrichten in kurzem Abstand werden durchgelassen
- [ ] Globale Memory "Agent Chat-Regeln" ist im Agent-Payload sichtbar
