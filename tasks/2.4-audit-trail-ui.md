# 2.4 Audit Trail UI

**Priorität:** MITTEL
**Aufwand:** 3 Tage
**Phase:** 2 — Workflow-Reife

## Ziel

Timeline-View im Frontend: Was ist in einem Projekt passiert? Wer hat was wann getan? Chronologisch, filterbar, durchsuchbar.

## Hintergrund

Audit Trail (Task 1.2) loggt alle Aktionen in der DB. Ohne UI sieht das nur ein Entwickler per SQL. Die GL will Transparenz — "zeigt mir was der Agent gemacht hat" muss mit einem Klick beantwortbar sein.

## Anforderungen

### Neue Seite: Projekt-Aktivitaet

Route: `/projects/{projectId}/activity`

Erreichbar ueber Tab im Projekt-Detail (neben Tasks, Chat, Attachments).

### Timeline-Darstellung

Chronologisch, neueste zuerst. Jeder Eintrag zeigt:

```
[Avatar] [Agent/User Name] — [Aktion] — [Zeitpunkt]
         [Details-Zeile]

Beispiele:
🌋 Lava — Task in_progress gesetzt — vor 3 Min
   "Screening-Kriterium: Risiken und Go/No-Go"

🌋 Lava — Attachment hochgeladen — vor 2 Min
   screening_review_nrw_ki.md (12 KB) → Task "Screening-Kriterium..."

🌋 Lava — Task done gesetzt — vor 1 Min
   "Screening-Kriterium: Risiken und Go/No-Go"

❌ Lava — Attachment lesen fehlgeschlagen — vor 5 Min
   Ausschreibung_NRW.pdf — "API key invalid"
```

### Filter

- Nach Agent/User
- Nach Action-Typ (task.*, attachment.*, message.*)
- Nach Zeitraum
- Nur Fehler anzeigen (success = false)

### Pagination

- 50 Eintraege pro Seite
- Infinite Scroll oder "Mehr laden" Button
