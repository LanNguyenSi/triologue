# 4.2 MCP Connection UI

**Priorität:** MITTEL
**Aufwand:** 1 Tag
**Phase:** 4 — Connector UI
**Abhängigkeit:** 3.5 (MCP Bridge), 4.0 (Connector Admin Page existiert)

## Ziel

Admin kann MCP-Server über die UI hinzufügen, testen und verwalten. Für interne/Legacy-Systeme die kein REST+OAuth haben.

## Hintergrund

MCP Bridge (3.5) kann bereits MCP-Server erreichen und Tools discovern. Admin-Endpoints existieren. Es fehlt die UI.

## Anforderungen

### MCP-Abschnitt auf Connector Admin Page

Erweitert `/admin/connectors` (4.0):

**"Custom Connectors (MCP)" Abschnitt:**

1. **Connection hinzufügen:**
   - Name (z.B. "Internes Wiki")
   - URL (z.B. `http://wiki-mcp:3000/mcp`)
   - API Key (optional, wird verschlüsselt gespeichert)
   - "Verbinden & Tools entdecken" Button

2. **Connection-Liste:**
   - Name, URL, Status (active/error)
   - Entdeckte Tools mit Name + Beschreibung
   - Letzter Health Check Zeitpunkt
   - "Neu scannen" Button (Tool Discovery)
   - "Löschen" Button

3. **Status-Anzeige:**
   - Grün: aktiv, letzer Health Check < 10min
   - Rot: error, Server nicht erreichbar
   - Anzahl entdeckter Tools

### API Endpoints (existieren in 3.5)

```
POST   /api/admin/mcp/connections        → Neue Connection
GET    /api/admin/mcp/connections        → Liste
POST   /api/admin/mcp/connections/:id/discover → Tool Discovery
DELETE /api/admin/mcp/connections/:id    → Löschen
```

## Akzeptanzkriterien

- [ ] Admin kann MCP-Server hinzufügen (Name + URL + optionaler Key)
- [ ] Tool Discovery zeigt entdeckte Tools an
- [ ] Status wird live angezeigt (aktiv/fehler)
- [ ] Admin kann Connection löschen
- [ ] API Key wird nie im Frontend angezeigt (nur "***" oder "Konfiguriert")
