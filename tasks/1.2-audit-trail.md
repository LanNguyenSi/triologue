# 1.2 Audit Trail

**Priorität:** KRITISCH
**Aufwand:** 3 Tage
**Phase:** 1 — Fundament

## Ziel

Jede Agent-Aktion wird in einer `agent_audit_log` Tabelle protokolliert. Wer hat was wann getan, war es erfolgreich, wie lange hat es gedauert.

## Hintergrund

Beim Pitch schlug die PDF-Analyse fehl, aber niemand sah den Fehler bis es zu spaet war. Mit Audit Trail haetten wir sofort gesehen: "Lava, attachment.read, Ausschreibung_NRW.pdf, FAILED, API key invalid". Audit Trails sind auch eine Compliance-Anforderung fuer Enterprise-Kunden.

## Anforderungen

### Prisma Schema

```prisma
model AgentAuditLog {
  id           String   @id @default(cuid())
  timestamp    DateTime @default(now())
  agentId      String                        // User-ID des Agents
  action       String                        // z.B. "task.claim", "attachment.upload"
  resourceType String                        // z.B. "task", "attachment", "message"
  resourceId   String?                       // ID des betroffenen Objekts
  projectId    String?
  roomId       String?
  details      Json     @default("{}")       // Kontext: alte/neue Werte, Fehler
  success      Boolean  @default(true)
  durationMs   Int?                          // Ausfuehrungsdauer
  
  agent        User     @relation(fields: [agentId], references: [id])
  
  @@index([agentId])
  @@index([projectId])
  @@index([timestamp])
  @@index([action])
  @@map("agent_audit_log")
}
```

### Zu loggende Actions

| Action | Trigger | Details |
|--------|---------|---------|
| `task.claim` | Agent setzt Status auf in_progress | taskId, title |
| `task.update` | Agent aendert Task-Felder | field, oldValue, newValue |
| `attachment.read` | Agent liest Attachment-Content | attachmentId, filename, mimeType |
| `attachment.upload` | Agent laedt Datei hoch | filename, mimeType, sizeBytes, taskId |
| `message.send` | Agent postet in Room | roomId, contentLength |
| `memory.read` | Agent ruft Memories ab | count, scope |
| `context.fetch` | Agent ruft Task Runtime Context ab | taskId |
| `screening.run` | Plugin startet Screening | runId, projectId, taskCount |

### Logging-Service

```typescript
// server/src/services/auditService.ts

interface AuditEntry {
  agentId: string;
  action: string;
  resourceType: string;
  resourceId?: string;
  projectId?: string;
  roomId?: string;
  details?: Record<string, unknown>;
  success?: boolean;
  durationMs?: number;
}

export async function logAuditEvent(entry: AuditEntry): Promise<void> {
  // Fire-and-forget: Audit Logging darf nie den Hauptflow blockieren
  prisma.agentAuditLog.create({ data: entry }).catch(err => {
    console.error('[audit] Failed to log:', err);
  });
}
```

### API Endpoint (Read)

```
GET /api/agents/audit?projectId={id}&limit=50&offset=0
GET /api/agents/audit?agentId={id}&action=attachment.upload
Authorization: Bearer <BYOA Token oder Admin>
```

Response:
```json
{
  "items": [
    {
      "timestamp": "2026-03-13T09:34:58Z",
      "agent": "lava",
      "action": "attachment.read",
      "resourceType": "attachment",
      "resourceId": "att-001",
      "success": false,
      "details": { "error": "API key invalid", "filename": "Ausschreibung_NRW.pdf" },
      "durationMs": 1240
    }
  ],
  "totalCount": 42
}
```

### Berechtigung

- Agents sehen nur eigene Audit-Eintraege
- Admins sehen alle
- Projekt-Mitglieder sehen Projekt-bezogene Eintraege
