# 1.5 Ice Session-Split

**Priorität:** HOCH
**Aufwand:** 1 Tag
**Phase:** 1 — Fundament

## Ziel

Ice bekommt eine separate Session fuer Triologue, damit Telegram nicht blockiert wird wenn Ice in Triologue aktiv ist (und umgekehrt).

## Hintergrund

Ice laeuft aktuell auf einer einzigen Session (`agent:main:main`). OpenClaw serialisiert Messages innerhalb einer Session — d.h. wenn Ice eine Triologue-Nachricht verarbeitet, wartet Telegram. Das fuehrte beim Pitch dazu, dass Ice nicht parallel reagieren konnte.

Lava hat dieses Problem nicht, weil sie ein separates Daemon-Setup nutzt (eigener Prozess fuer Triologue).

## Anforderungen

### SSE Client Konfiguration aendern

Aktuell in `ice-triologue-sse.service`:
```
Session: agent:main:main
```

Aendern zu:
```
Session: agent:triologue
```

### OpenClaw Konfiguration

In `openclaw.json` pruefen ob eine zweite Session konfiguriert werden muss oder ob OpenClaw automatisch neue Sessions akzeptiert.

### Was sich aendert

| Kanal | Session | Verarbeitung |
|-------|---------|-------------|
| Telegram | agent:main:main | Wie bisher |
| Triologue | agent:triologue | Neue, separate Session |

### Was gleich bleibt

- Beide Sessions teilen denselben Workspace (MEMORY.md, memory/, etc.)
- Beide Sessions nutzen dasselbe Modell
- Ice ist ein Agent mit zwei Eingangskanaelen, nicht zwei Agents

### Risiken

1. **Geteilter Kontext:** Wenn Ice in Triologue etwas lernt, weiss die Telegram-Session nichts davon (bis zum naechsten Memory-Read). Das ist akzeptabel — Memory Weaver soll das langfristig loesen.
2. **Doppelte Antworten:** Wenn eine Nachricht ueber beide Kanaele kommt, koennte Ice doppelt antworten. Unwahrscheinlich, da Triologue und Telegram verschiedene Quellen sind.

## Dateien die geaendert werden muessen

1. **`/etc/systemd/system/ice-triologue-sse.service`** — Session-Variable aendern
2. **`triologue-agent-gateway/examples/openclaw-sse-client.ts`** — Session aus ENV lesen (falls hardcoded)
3. **`/root/.openclaw/openclaw.json`** — Pruefen ob Session-Config noetig

## Tests

- [ ] Ice antwortet in Telegram waehrend Triologue-Nachricht verarbeitet wird
- [ ] Ice antwortet in Triologue waehrend Telegram-Nachricht verarbeitet wird
- [ ] Beide Sessions lesen dieselben Workspace-Dateien
- [ ] Kein Crosstalk (Telegram-Antwort landet nicht in Triologue)
